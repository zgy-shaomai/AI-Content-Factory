## 1. 项目周期与里程碑

### 1.1 总工期

**20 - 30 个工作日**（自客户提供完整资料 + API 凭据 + VPS 当日起算）。

### 1.2 阶段划分

| 阶段 | 工作日 | 主要工作 | 交付物 |
|---|---|---|---|
| 阶段 A：底座搭建 | 第 1-5 工作日 | VPS 部署、Postgres / N8N / Caddy / Redis 上线、OSS 桶与生命周期配置、飞书多维表初始化 | 系统可访问、N8N 可登录、空 SKU 可建任务 |
| 阶段 B：图片链路打通 | 第 6-12 工作日 | 围绕样品 SKU 跑通"输入 → prompt 生成 → 图片生成 → 候选回写 → 审核 → 归档" | 单 SKU 图片链路一键可跑 |
| 阶段 C：视频链路打通 | 第 13-20 工作日 | 分镜脚本生成、Seedance 2.0 联调、ASR 字幕、FFmpeg 合成、归档 | 单 SKU 视频链路一键可跑 |
| 阶段 D：调优与验收 | 第 21-25 工作日 | 围绕样品做效果调优、模板抽象、多 SKU 复用验证、文档定稿 | 验收 demo 录屏、SOP、模板交接 |
| 缓冲期（可选） | 第 26-30 工作日 | 处理验收意见、做最后一轮收口 | 验收报告 |

如客户在阶段 A 之前未能完成"客户配合 checklist"中的关键项（API 凭据、VPS、飞书机器人权限），整体工期顺延，不计入承诺工期内。

### 1.3 保修期

**交付验收完成日起 1 个月**，详见 §6。

## 2. 客户提交清单（Inputs）

以下材料 / 凭据 **由客户在开工前 5 个工作日内** 提供齐全。任何一项延迟将延后开工日期。

### 2.1 产品资料

- 首期核心样品 SKU 的完整资料：
  - 产品命名、型号编号
  - 详细参数表（面料成分、尺码区间、颜色色号、克重等）
  - 卖点描述（3-5 条核心卖点的原文文案）
  - 产品白底图（高清 PNG，正面 / 侧面 / 背面 / 细节）
  - 产品穿着实拍图（如有）
- 后续多产品复用所需的字段口径模板（首期跑通后由乙方整理，客户确认）。

### 2.2 参考素材

- 风格参考图（5-15 张，标注每张希望参考什么：构图 / 光线 / 模特 / 色调）
- 对标视频（3-10 条，标注希望参考的节奏 / 镜头 / 文案）
- 不希望出现的反例（≥ 3 张，避免 AI 重蹈覆辙）

### 2.3 第三方 API 凭据

- **5dock NewAPI**：API Key + Base URL（用于图片 / LLM 模型调用）
- **火山引擎方舟**：API Key + Region + Endpoint（用于 Seedance 2.0 视频生成）
- **阿里云 OSS**：Access Key ID / Secret + Bucket 名 + Endpoint（已建好 3 个 Bucket）
- **飞书开放平台**：App ID + App Secret + 多维表 token + 机器人 webhook URL
- **ASR 服务**：（任一）阿里云语音、火山 ASR、Whisper 自建 — 凭据按所选方案给

### 2.4 VPS 与域名

- Ubuntu 22.04 LTS、4C8G、≥ 100GB SSD 的 VPS 1 台
- root 或 sudo 用户的 SSH 凭据
- 一级域名 1 个 + 两个子域名解析（如 `n8n.example.com` / `api.example.com`）→ 解析到 VPS 公网 IP
- 安全组 / 防火墙开放：22 / 80 / 443

### 2.5 验收与审核口径

- 客户侧的审核负责人 1 人 + 审核员 1-3 人（在飞书空间内开账号 + 加入审核多维表）
- 客户侧明确"商用品质判定标准"的书面口径（避免后期"我觉得不行"无法量化）
- 客户侧确认"是否需要 PS 人工精修"——本项目**不含**人工 PS，如需要由客户另行采购

## 4. 验收清单（Acceptance Criteria）

> 每条直接对应需求原文"验收与复用口径"和各模块描述。

| 编号 | 验收项 | 验收方法 | 通过判据 |
|---|---|---|---|
| AC-01 | 多产品共用的输入字段结构 | 在飞书多维表为 SKU `YN-BRA-001` 完整填写所有字段 | 字段齐全、保存无报错、N8N 收到 webhook |
| AC-02 | 图片链路跑通 | 提交一次任务，要求生成 4 张候选商品图 | 4 张候选在 5 分钟内全部回写到飞书并可预览 |
| AC-03 | 视频链路跑通 | 提交一次任务，生成 1 条 10-15 秒的视频 | 成片含原生音频 + 字幕，时长 10-15 秒，回写飞书可预览 |
| AC-04 | 提示词生成规则 | 同一 SKU 的 prompt 应可在审核时被驳回类别触发改写 | 二次生成后新 prompt 与旧 prompt 在 LLM 比较中"针对性修改"得分 > 80% |
| AC-05 | 风格与一致性控制 | 4 张候选图之间风格一致（同 SKU、同色调、同光线方向） | 客户审核员目测 ≥ 3 / 4 张达标 |
| AC-06 | 审核回写机制 | 在飞书改"审核状态"，N8N 应在 30 秒内执行对应动作 | 通过 → 归档；二次生成 → 新候选生成；打回 → 写状态 |
| AC-07 | 素材归档与任务流转 | OSS 路径与 Postgres `archive` 表一一对应；交付包可打 zip | OSS Key 数 == archive 行数 == zip 内文件数 |
| AC-08 | 首期样板验收 | 围绕样品 SKU 完成首轮联调 | 至少有 1 组（4 图 + 1 视频）通过审核并入交付路径 |
| AC-09 | 多产品复用能力 | 现场新增一个 SKU（不需改代码） | 新 SKU 能进入工作流并生成候选物 |
| AC-10 | 商用品质口径 | "甲方提供样本，工作流产出效果一样或者更高" | 客户对比验收 demo 与对标样本，给出书面通过 |
| AC-11 | 异常处理 | 模拟 API 超时、模型返回失败 | 系统自动重试 3 次、最终失败时写状态 + 告警飞书群 |
| AC-12 | SLA 监控 | 制造一个超时未审任务（>4h） | 飞书告警群收到推送 |

**验收方式**：分两轮。第一轮 demo 验收（验证可跑），第二轮使用验收（客户实际跑 5-10 个 SKU 一周内不出阻塞性 bug）。两轮都通过即视为最终验收。

## 6. 保修期范围（1 个月免费保修）

### 6.1 保修期起算

最终验收报告签字 / 客户书面确认通过日的次日 0:00。

### 6.2 包含

- **Bug 修复**：交付范围内出现的功能性 bug（不能跑、跑错、丢数据），1 个工作日内响应、5 个工作日内修复。
- **配置维护**：因第三方 API 凭据更新（仅限同一服务的 Key 轮换）、域名 SSL 续签等导致的配置调整，免费支持。
- **使用答疑**：在工作日 9:00-18:00 内通过飞书群 / 邮件答疑，最长 1 个工作日内回复。
- **小幅调优**：单次工作量 < 0.5 工作日的 prompt 微调、阈值微调，免费支持，每月累计 ≤ 3 工作日。

### 6.3 不包含

- 新功能开发、新 SKU 模板适配（按需评估）
- 跨品类（非服装）扩展
- 新平台、新模型的接入
- 客户更换 VPS / 更换 OSS 区域 / 更换飞书账号 引发的整体迁移
- 第三方服务（5dock、火山、OSS、飞书）侧的故障导致的不可用
- 客户业务逻辑变更引起的重写

保修期内累计 > 3 工作日的"小幅调优"或任何超出保修范围的需求，按 0.5 - 1 工作日为单位另行计费。

## 7. 不在范围内的事（Out of Scope）

> 以下条目按需求原文"默认排除范围"扩展具体例子。客户如需以下任何一项，需另行评估工期与费用。

### 7.1 跨品类大规模批量稳定化

- ❌ 接入家纺、3C、美妆等非服装品类（每个品类需单独训练 prompt 模板与风格规则）
- ❌ 同时跑 100+ SKU 持续 7×24 不间断生产并保证 99% 通过率
- ❌ 复杂面料（如蕾丝、亮片、印花叠加）的精细纹理还原承诺

### 7.2 长期代运营

- ❌ 由乙方人工每天值守审核
- ❌ 由乙方运营客户的飞书多维表 / 监控告警
- ❌ 由乙方负责持续刷量、调优 prompt、迭代风格

### 7.3 无限次返工或无限制效果调优

- ❌ 同一候选物超过 3 轮仍要求"再来一张"——超出按 prompt 微调单独计时
- ❌ 客户审美口径反复变更引发的全套模板重写
- ❌ "我觉得这张不够好"但说不出具体改向的退回（必须按驳回类别 + 改向描述结构化）

### 7.4 人工 PS 服务

- ❌ Photoshop / Lightroom / Procreate 等手工修图
- ❌ Blender / C4D 三维场景搭建
- ❌ 视频后期剪辑（剪映 / Pr / DaVinci 操作）
- ❌ 人物面部精修、皮肤重绘
- 注：如客户需要，乙方可推荐外部供应商，但服务与费用与本项目无关。

### 7.5 大量定制前端系统开发

- ❌ 独立 Web 后台（React / Vue 自建）
- ❌ 移动端 App
- ❌ BI 大屏 / 自定义可视化
- ❌ 多角色权限系统（本项目仅做 飞书账号 + 多维表权限两层）
- ❌ 客户品牌定制 UI

### 7.6 其他（按 PDF 原文兜底）

- 新增功能、需求变更、范围扩展、多品类大规模扩展、新增复杂人工精修要求 — 均需另行评估。

## 8. 复用口径（Reusability）

- 本次交付重点为"**可复用的内容工厂模板能力**"，不是只交付单个产品的一次性内容产出。
- 复用范围：**服装类目**。
- 同类目新增 SKU 不收开发费（仅需客户填表 + 提供新产品资料），但若新增需求超出本次模板能力（如新增视频时长、新增模型、新增审核流），按工作量评估。
- 不在复用范围内的：跨品类、大规模批量稳定化、新增复杂要求。
