# 项目文档导航

本目录用于集中存放项目设计说明。阅读这些文档的目标不是记住所有细节，而是快速建立对系统结构、链路、审核和归档的整体理解。

## 1. 推荐阅读顺序

如果你第一次接手这个项目，推荐按下面顺序阅读：

1. 根目录 `README.md`
2. 根目录 `SETUP.md`
3. `docs/architecture.md`
4. `schemas/README.md`
5. 按需阅读图片、视频、审核、归档文档

## 2. 各文档作用

### 2.1 `architecture.md`

这是总览文档。

适合解决的问题：

- 这个项目整体是怎么组织的
- 工作流、数据库、存储、审核是怎么连起来的
- 一条 SKU 从录入到归档大致经过哪些步骤
- 为什么图片链路和视频链路能共用一套资料

如果你只能先看一篇文档，优先看这一篇。

### 2.2 `image-pipeline.md`

这是图片链路的细节文档。

适合解决的问题：

- 图片链路每一步的输入输出是什么
- prompt 是怎么组装的
- Seedream 的参数怎么设计
- reference image 怎么参与生成
- 图片候选如何写回数据库和审核台

### 2.3 `video-pipeline.md`

这是视频链路的细节文档。

适合解决的问题：

- 视频链路如何从商品资料走到分镜、关键帧和最终成片
- Seedance 调用策略是什么
- 字幕如何生成和烧录
- 视频链路与图片链路共用了哪些信息

### 2.4 `audit-workflow.md`

这是审核工作台设计文档。

适合解决的问题：

- 审核员如何做通过、打回、二次生成
- 审核动作如何回写到工作流
- 为什么反馈要结构化
- SLA 和告警如何设计

### 2.5 `archive-structure.md`

这是归档规范文档。

适合解决的问题：

- OSS bucket 如何命名
- 原始资料、候选物、交付物如何分目录
- 交付包如何组织
- 为什么归档路径要能和数据库一一对应

## 3. 怎么根据问题找文档

如果你遇到的是下面这些问题，可以直接定位：

| 问题 | 优先看 |
|---|---|
| 项目整体怎么运作 | `architecture.md` |
| 图片为什么没生成出来 | `image-pipeline.md` |
| 视频为什么没成片 | `video-pipeline.md` |
| 审核状态为什么没回写 | `audit-workflow.md` |
| OSS 路径怎么设计 | `archive-structure.md` |
| 表结构、状态机、字段定义 | `schemas/README.md` 与 `schemas/` 下文档 |

## 4. 与其他目录的关系

`docs/` 是解释层，其他目录是执行层：

- `n8n/`：工作流定义
- `schemas/`：数据契约
- `prompts/`：提示词模板
- `scripts/`：联调脚本
- `deploy/`：运行环境

最常见的配合方式是：

1. 先在 `docs/` 里理解设计
2. 再去 `n8n/` 或 `scripts/` 看真实实现
3. 最后回到 `schemas/` 对齐字段和状态

## 5. 阅读建议

阅读这些文档时，建议重点抓住三条主线：

1. 输入资料从哪里来
2. 状态如何流转
3. 资产最后如何归档

只要这三条线清楚了，项目的大部分结构都会顺下来。
